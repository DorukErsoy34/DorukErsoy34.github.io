1.5 
Casting and Range of Variables
Casting changes a variable‚Äôs type, e.g., turning a string into a number.
Example: Number("42") converts the string "42" to the number 42.
Range refers to the limits of values a type can hold (like how integers have a max size).
Quick takeaway: Casting ensures data is in the right form for calculations.
1.6 
Compound Assignment Operators
Shorthand operators update variables more quickly.
Example: x += 5 means x = x + 5; x *= 2 means x = x * 2.
Quick takeaway: Compound operators save time when updating variable values.
1.7 
Application Program Interface (API) and Libraries
APIs and libraries give access to ready-made functions.
Example: JavaScript‚Äôs Math library has functions like Math.sqrt(25) or Math.random().
Quick takeaway: APIs extend your language with powerful tools you don‚Äôt need to build yourself.
1.8
Documentation with Comments
Comments explain code for humans, ignored by the computer.
In JavaScript, // makes a single-line comment and /* ... */ makes a multi-line comment.
Quick takeaway: Comments make your code readable and maintainable.
1.9 
Method Signatures
A method (function) signature shows its name, parameters, and return type.
Example in JavaScript: function greet(name) { return "Hello " + name; } has name = greet, parameter = name, return = string.
Quick takeaway: Method signatures define how a function is used.
1.10 
Calling Class Methods
Class methods (also called static methods) belong to the class, not to an individual object. You call them using the class name.
Example: int max = Math.max(5, 10);
Quick takeaway: Class methods are called with the class name and don‚Äôt need an object.
1.11 
Math Class
The Math class has many built-in static methods for calculations.
Example: Math.sqrt(25) ‚Üí 5.0, Math.random() ‚Üí random number between 0.0 and 1.0
Quick takeaway: The Math class provides ready-to-use formulas and functions.
1.12
Objects: Instances of Classes
An object is an instance of a class, combining data (attributes) and behaviors (methods).
Example: String word = new String("Hello");
Quick takeaway: Objects are the ‚Äúreal things‚Äù created from class blueprints.
1.13 
Object Creation and Storage (Instantiation)
You create an object using the new keyword and store it in a variable.
Example: Scanner input = new Scanner(System.in);
Quick takeaway: Instantiation sets up a new object in memory using a constructor.
1.14
Calling Instance Methods
Instance methods belong to objects and can only be called on a specific instance.
Example: "Hello".toUpperCase() ‚Üí "HELLO"
Quick takeaway: Instance methods work with the data of a particular object.
1.15 
String Manipulation
Strings are objects with methods for handling text.
Example: "Hello".substring(0, 2) ‚Üí "He", "Hello".length() ‚Üí 5
Quick takeaway: String methods make it easy to analyze and transform text.

üîπ Repeat Strings Multiple Times
public String repeat(String word, int count) {
    String result = "";
    for (int i = 0; i < count; i++) {
        result += word;
    }
    return result;
}

// Example calls:

UNIT 2:

üîÅ Iteration
Iteration means repeating a section of code multiple times.
In Java, this is done using loops such as:
for loops (when you know how many times to repeat)
while loops (when you repeat until a condition becomes false)
Example:
for (int i = 0; i < 5; i++) {
    System.out.println("Hello!");
}
This prints ‚ÄúHello!‚Äù five times.

‚öôÔ∏è Selection
Selection is when a program makes decisions based on conditions.
It allows the program to choose different paths of execution.
The most common selection structure is the if-else statement.
Example:
if (score >= 10) {
    System.out.println("You passed!");
} else {
    System.out.println("Try again!");
}
Here, the output depends on whether the score condition is true or false.

üß© Boolean Expressions
A boolean is a data type that can hold only two values: true or false.
Boolean expressions are used in decision-making and control structures.
Example:
boolean isEven = (number % 2 == 0);
System.out.println(isEven);
This prints true if the number is even, or false otherwise.

üîÑ If-Else Statements
The if-else structure is a key part of algorithm design because it introduces logic and flexibility.
It checks a condition, and if it‚Äôs true, it executes one block of code; otherwise, it executes another.

Example:
if (temperature > 30) {
    System.out.println("It's hot outside!");
} else if (temperature >= 20) {
    System.out.println("It's warm.");
} else {
    System.out.println("It's cold!");
}
This uses multiple conditions to select one of several possible outcomes.

2.4 Nested if Statements
An if statement inside another if (or else).
Used when a decision depends on another condition.
Example:
if (condition1) {
    if (condition2) {
        // both true
    } else {
        // condition1 true, condition2 false
    }
} else {
    // condition1 false
}
Nesting too deep can reduce readability.

2.5 Compound Boolean Expressions
Combine multiple boolean conditions using logical operators:
AND && ‚Üí true if both are true
OR || ‚Üí true if at least one is true
NOT ! ‚Üí negates a boolean
Example:
boolean canRide = age >= 12 && height >= 48;
boolean alert = !isSafe || hasWarning;
Parentheses help clarify order of operations.
    
2.6 Comparing Boolean Expressions
Check if boolean expressions produce the same result.
== compares primitive values.
.equals() compares object contents.
Example:
boolean a = true;
boolean b = (5 > 3);
System.out.println(a == b); // true
For objects like Strings, == checks reference; use .equals() for content comparison.

2.7 While Loops
Definition: A while loop repeatedly executes a block of code as long as a condition is true.
Syntax:
while (condition) {
    // code to repeat
}
Key Points:
Condition is checked before the loop body runs (pre-test loop).
Infinite loop risk if condition never becomes false.
Can be used for unknown number of iterations.
Example:
int count = 0;
while (count < 5) {
    System.out.println(count);
    count++;
}

2.8 For Loops
Definition: A for loop is used when the number of iterations is known or predictable.
Syntax:
for (initialization; condition; update) {
    // code to repeat
}
Key Points:
Initialization: run once at the start.
Condition: checked before each iteration.
Update: executed after each iteration.
Can count up or down, or even use multiplication/division.
Example:
for (int i = 0; i < 5; i++) {
    System.out.println(i);
}

2.9 Implementing Selection and Iteration Algorithms
Selection (Decision Making)
Purpose: Choose between alternatives using if, else if, else, or switch.
Example:
int num = 10;
if (num > 0) {
    System.out.println("Positive");
} else if (num < 0) {
    System.out.println("Negative");
} else {
    System.out.println("Zero");
}
Iteration (Repeating Tasks)
Purpose: Repeat tasks using loops (while, for, do-while).
Common Patterns:
Counter-controlled iteration: repeat N times (use for).
Sentinel-controlled iteration: repeat until a special value is entered (use while).
Example (Counter-controlled):
for (int i = 1; i <= 5; i++) {
    System.out.println("Iteration " + i);
}
Example (Sentinel-controlled):
Scanner input = new Scanner(System.in);
int value = 0;
while (value != -1) {
    System.out.print("Enter a number (-1 to stop): ");
    value = input.nextInt();
}
2.10 Implementing String Algorithms
String algorithms involve processing or analyzing text, e.g., searching, replacing, counting, or pattern matching.
Common operations:
charAt(index) ‚Üí get character at position
length() ‚Üí number of characters
substring(start, end) ‚Üí extract part of a string
indexOf(substring) ‚Üí find position of substring
equals() ‚Üí compare strings
Example: Count vowels or check if a substring exists in a string.
Efficient string algorithms avoid repeated concatenation or unnecessary loops.
2.11 Nested Iteration
Nested iteration = a loop inside another loop.
Outer loop controls overall repetitions, inner loop handles sub-tasks.
Commonly used for:
2D arrays or matrices
Printing patterns (e.g., triangles, squares)
String processing (looping over each character in multiple strings)
Example:
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        System.out.println(i + "," + j);
    }
}
2.12 Informal Run-Time Analysis
Estimates how long an algorithm takes without precise formulas.
Focus: How time grows as input size grows (efficiency).
Common patterns:
Loops ‚Üí linear (O(n)), nested loops ‚Üí quadratic (O(n^2))
Simple operations inside loops ‚Üí constant time (O(1))
Helps compare algorithms and anticipate performance problems.
Example: Counting vowels in a string of length n ‚Üí O(n) time. Nested loop over n x n ‚Üí O(n^2).
